Security enforced by the platform, not by user discipline

Security that depends on people remembering to be careful is not security. In Ospia the controls are structural — enforced by the architecture and checked by the build pipeline.

Isolation

Your data is not in a shared table

Multi-tenancy is physical, not logical. There is no query that could return another hospital's data, because another hospital's data is not in your database.

Tenant isolation

One physical database per hospital

Not a tenant column, not a schema flag — a separate database. Self-hosted or managed cloud, your choice, and the choice is reversible.

physical isolation · self-host or managed · portable
Row-level security

Scoped at the database

Postgres row-level security applies hospital and user context to every query, beneath the application layer, so an application bug cannot bypass it.

enforced below the code that could get it wrong
Field encryption

Personal data encrypted at rest

AES-256-GCM on personal data with separate keys for audit and PII, and a secrets provider abstraction with rotation hooks.

no secrets in the repository, ever
Access control

RBAC plus ABAC

Permissions evaluated with context — assignment, admission state, ward, shift, time — so "a doctor may edit only admitted patients assigned to them today" is a policy expression, not code.

context-aware authorization, matrix-tested in CI
Audit chain

Tamper-evident by construction

Every mutation is hash-chained. Edit one historical entry and the whole chain fails verification — which is the point.

re-verifiable end to end, on demand
Patient plane

Separate identity plane

Patients authenticate on a cryptographically separate plane; a portal token is useless against staff APIs.

no shared session surface between patients and staff

Programme

Security as a pipeline, not a policy

Controls that are not tested are aspirations. These run on every change.

Dependency scanning

Every dependency checked for known vulnerabilities on each build; failures block the merge.

Secret scanning

Credentials committed to the repository fail the build. Secrets live in a vault provider with rotation.

Authorization matrix

Every endpoint tested against every role, so a permission regression cannot ship silently.

Security event pipeline

Auth failures, permission denials, unmasked-PII reads and behavioural anomalies emitted in SIEM-consumable form.

Backup & recovery

Documented recovery objectives with restoration rehearsed rather than assumed.

Disclosure

A published route for reporting vulnerabilities, and a commitment to tell affected hospitals promptly.

Formal certification status is stated plainly on our compliance page — including what we have, what is in progress, and what we have not started. We will not describe a roadmap item as a certification.
